# AI Prompt Log

> AI(Claude Code)를 활용한 코드 수정 과정을 기록한 문서입니다.
> 어떤 프롬프트를 입력했고, 어떤 결과가 나왔는지 참고하세요.

## 읽는 법

각 항목은 아래 구조로 되어 있습니다:

| 항목 | 설명 |
|------|------|
| **프롬프트** | AI에게 입력한 지시문 (원문 그대로) |
| **의도** | 이 프롬프트를 친 이유 / 해결하려는 문제 |
| **수정된 파일** | AI가 변경한 파일 목록 |
| **결과 요약** | 실제로 어떤 변경이 이루어졌는지 |
| **배운 점** | 프롬프트 작성 시 알아두면 좋은 팁 |

## 프롬프트 기록

---

### #1. 프로젝트 구조 분석 및 리뷰 문서 생성

**프롬프트:**
```
지금 백엔드, 프론트엔드의 구조를 보고 한 번 어떤걸 고쳐야할지 확인해볼래??
앱을 만든다고 하던데
```

```
이 내용을 좀 md로 넣어서 학생들이 보고 참고할 수 있도록 해줄래??
```

**의도:** 학생들이 만든 프로젝트의 현재 상태를 파악하고, 어떤 부분이 부족한지 정리해서 학생들에게 전달하기 위함

**수정된 파일:**
- `PROJECT_REVIEW.md` (신규 생성) - 프로젝트 분석 결과 및 개선 방향 정리

**결과 요약:**
AI가 프론트엔드/백엔드 전체 파일을 읽고 분석한 뒤, 현재 구현 상태, 부족한 부분, React Native 개선 방향, 우선순위 로드맵을 문서로 정리

**배운 점:**
- "구조를 보고 고칠 점 확인해줘"처럼 **열린 질문**을 하면 AI가 전체를 스캔해서 종합적인 분석을 해줌
- 분석 결과를 바로 코드에 반영하지 않고, 먼저 **md 문서로 정리**해달라고 하면 팀원과 공유하기 좋음
- 한 번에 "분석 + 문서화"를 시키기보다, 먼저 분석 → 결과 확인 → 문서화 순으로 **단계를 나누면** 결과 품질이 높아짐

### #2. 제품 기획 질문지 생성

**프롬프트:**
```
그리고 제품쪽으로도 부족한 내용을 학생들이 답하고 생각할 수 있도록 질문지도 만들어줘.
아마 그 질문지를 답으로 AI에게 다시 넣어서 개발할거니까
```

**의도:** 학생들이 기획 단계에서 빠뜨린 내용을 스스로 채우고, 그 답변을 AI 프롬프트로 재활용할 수 있게 하기 위함

**수정된 파일:**
- `PRODUCT_QNA.md` (신규 생성) - 8개 영역 40개+ 질문, AI에게 넘기는 방법 가이드 포함

**결과 요약:**
유저, 상품, 지도, 미션, 출석/크레딧, 채팅, 랭킹, 기술/배포 총 8개 영역에 대한 질문지를 생성. 각 질문 아래에 "답변:" 란을 두어 학생들이 바로 채울 수 있게 하고, 마지막에 "답변을 AI에게 넘기는 프롬프트 예시"까지 포함

**배운 점:**
- **AI의 출력이 다른 AI의 입력이 되는 구조**를 설계하면, 작업 흐름이 자연스러워짐
- "질문지를 만들어줘"라고만 하면 일반적인 질문이 나올 수 있으니, **"AI에게 다시 넣을 거다"라는 맥락**을 알려주면 더 실용적인 결과가 나옴
- 프롬프트에 **용도와 목적**을 함께 알려주는 게 핵심
